Useful SQLite "dot" Commands

In the sqlite3 command line interface "dot" commands (i.e. commands that start with a dot) are not SQL statements but rather special instructions for the sqlite3 shell. These dot commands do not end in a semicolon the way SQL stattements do. You can see a list of all the available shell commands by executing .help at the sqlite> prompt.

Note sqlite3 is the product we will be using. sqlite3 is a long term product and should not be considered a version number. Thus the application we will launch will always be sqlite3 and not sqlite or sqlite4, for example.

Some dot commands we will use a lot:

Command Explanation
.open

Open a database. You can specify the database to open when you execute sqlite3 command line or launch without a command line argument and then use .open from within sqlite3 application.

sqlite3 mydatabase.db

OR
sqlite3
.open mydatabase.db
.help List summary of all the dot commands.

.exit
.quit

Exit or quit the application
.tables List all the tables in the open database.
.schema TABLE Display schema definition of table TABLE in SQL
.schema Without a table name will display the schema for all your tables in the database
.schema sqlite_master

An sqlite database has a hidden table called sqlite_master that shows information about the database's tables. It can be helpul to query this.

sqlite> .schema sqlite_master
CREATE TABLE sqlite_master (
  type text,
  name text,
  tbl_name text,
  rootpage integer,
  sql text
);
sqlite>
.read FILE Read sql and cli commands from a file called FILE (e.g. telephone_switch.sql) in the current directory
.mode column Show output of SQL SELECT statements as columns.
Other modes:
  • csv - Comma-separated values column
  • column - Columns (left aligned)
  • html - HTML table markup
  • insert - SQL insert statements for TABLE
  • line - One value per line
  • list - Values delimited by .separator string
  • tabs - Tab-separated values
  • tcl - TCL list elements
.header ON|OFF Show column names header with output
.echo ON|OFF Show the sql query being processed (Helpful if producing output for an assignment or demonstration.)
.import FILE TABLE Import data from FILE (csv format) into TABLE. Helpful if you want to populate a database from spreadsheet, or similar, data.

.separator STRING

.separator COL ?ROW?

Set column separator used by .output and .import to STRING

Change the column separator, and optionally the row
separator, for both the output mode and .import
.show

Show values of various variables including the current separator string

	  sqlite> .show
        echo: off
         eqp: off
     explain: auto
     headers: on
        mode: column
   nullvalue: ""
      output: stdout
colseparator: "|"
rowseparator: "\n"
       stats: off
       width: 
    filename: :memory:
.output FILE Redirect output to file FILE.
.output stdout Redirect output back to stdout (console)
.dump TABLE Output the contents of table TABLE, or entire database, as an sql script
.width 10 20 5 Set the output column width for column mode. In this example the column widths of the three columns of output would be 10,20 and 5 characters.
